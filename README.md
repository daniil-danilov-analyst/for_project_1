# A/B-тестирование изменения стоимости премиум-подписки

## Описание проекта

В рамках pet-проекта проведён A/B-тест, целью которого была проверка влияния изменения стоимости премиум-подписки (при покупке через две новые платёжные системы) на ключевые метрики продукта. Эксперимент проводился на новых пользователях из нескольких стран.

## Цель теста

Определить, является ли эксперимент успешным с точки зрения:
- выручки на пользователя (ARPU),
- конверсии в платящих пользователей,
- структуры покупаемых типов подписок,
- удержания (retention) и других поведенческих метрик.

## Данные

В проекте используются 6 CSV-файлов:

- **users_control_1.csv**, **users_control_2.csv**, **users_test.csv** – информация о пользователях каждой группы.
- **transactions_control_1.csv**, **transactions_control_2.csv**, **transactions_test.csv** – информация о платежах пользователей.

Поля пользователей: `uid`, `age`, `attraction_coeff`, `coins`, `country`, `visit_days`, `gender`, `age_filter_start`, `age_filter_end`, `views_count`, `was_premium`, `is_premium`, `total_revenue`.

Поля транзакций: `uid`, `country`, `joined_at`, `paid_at`, `revenue`, `payment_id`, `from_page`, `product_type`.

## Этапы анализа

1. **Предобработка данных**
   - Замена пропусков (`coins`, `was_premium`, `is_premium` → 0, `visit_days` → пустая строка).
   - Преобразование типов (даты, логические поля).
   - Удаление дубликатов транзакций.
   - Исключение логически некорректных транзакций (дата оплаты раньше регистрации).
   - Корректировка `total_revenue` и статуса `is_premium` на основе очищенных транзакций.

2. **Проверка корректности сплитования**
   - Сравнение контрольных групп между собой по:
     - конверсии в премиум,
     - распределению по странам,
     - возрасту,
     - коэффициенту привлекательности,
     - количеству дней посещения,
     - выручке.
   - Объединение контрольных групп в одну для увеличения мощности тестов.

3. **Анализ результатов**
   - **ARPU** – с помощью теста Манна‑Уитни и бутстрапа.
   - **Доля платящих пользователей** – логистическая регрессия.
   - **Retention** – визуализация и тесты для разных групп.
   - **Структура покупок** – анализ типов подписок (`trial_premium` vs `premium_no_trial`).
   - **Выручка с платящего пользователя (ARPPU)** – для подписки без пробного периода.

## Используемые технологии и библиотеки

- `pandas` – обработка данных
- `numpy` – вычисления
- `scipy.stats` – статистические тесты (Манна‑Уитни, хи‑квадрат, Краскела‑Уоллиса)
- `pingouin` – дополнительные тесты (при необходимости)
- `statsmodels` – логистическая регрессия
- `matplotlib` и `seaborn` – визуализация

## Основные результаты

1. **Контрольные группы статистически не различаются** по всем проверенным характеристикам – сплитование выполнено корректно.

2. **Конверсия в премиум** (как общая, так и в подписку без пробного периода) **не изменилась** статистически значимо. Однако доля платящих пользователей (любые транзакции) **снизилась на ~21,5%** (логистическая регрессия, p-value < 0.05).

3. **ARPU** (выручка на пользователя):
   - При полных данных значимого различия не обнаружено.
   - После отсечения выбросов (98‑й процентиль) выявлено **статистически значимое снижение** ARPU в тестовой группе.

4. **ARPPU** (выручка на платящего пользователя):
   - Для всех платящих – различий нет.
   - Для платящих с типом `premium_no_trial` – **выручка значимо выросла** (что естественно при повышении цены).

5. **Retention** (количество дней посещения) не изменился между группами.

6. **Структура покупок**: в тестовой группе пользователи чаще выбирали пробную подписку (`trial_premium`) вместо `premium_no_trial` (p-value < 0.05).

## Выводы и рекомендации

- Повышение стоимости **увеличило ARPPU** для тех, кто всё же купил платную подписку без пробного периода.
- Однако **общее число платящих пользователей снизилось**, что привело к **падению ARPU**.
- Пользователи переориентировались на пробную подписку, после которой могут отказаться от дальнейших платежей.
- **Эксперимент не может считаться однозначно успешным** с точки зрения роста выручки. Для внедрения нового тарифа рекомендуется:
  - Дополнительное тестирование с другими сегментами.
  - Введение новых функций или типов подписок для стимулирования покупок.
  - Анализ долгосрочного LTV после пробного периода.
